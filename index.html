The project

In this project I firstly extracted all the data from the RateYourMusic.com Website. RateYourMusic is a large database in which users are able to insert musical records as entries and to label them accordingly, for instance setting the musical genres, a range of descriptors and so on. Starting from an unstructured dataset has many advantages. First of all, we can exploit a larger amount of data that has been gathered, organized and labeled throughtout many years by the userbase of the website. Secondary, it potentially allows to exploit other interesting info that are not usually available in likned dataset such as DBpedia or Wikidate, for instance the pondarate average rating that is given from users. This could lead to a system which not only automatically predic the genres, descriptors and features of an album records, but also the probability of its likeness

Then a simple and small ontology was developed using Protegé as a software. The ontology links an album record with its main properties, namely the artist that made the record, its main genre, its secondary genre, some descriptors and so on. Moreover, since one of the goals was to allign the dataset with Wikidata, the artist subclasses was subdivded in two different classes, i.e. humans (songwriters, performers, musicians, etc.) and musical groups. Those subclasses have properties as well, such as a birthplace for humans, a list of members for bands and so on. The list of properties for artists was completely arbitrary and it can be extended any time. You can download the Protegé file here.

The steps

Lorem ipsum dolor sit amet consectetur.

  • Gather raw data

    The data were extracted from the charts using a JavaScript application developed by Simone del Deo which converts them into a raw .csv file.

  • Clean the data and link them to Wikidata

    In order to create a dataset semantically meaningful, some basic cleaning was requried. Pandas was used to eliminate duplicates, excluding useless columns and linking each entry with the realted Wikidata entry.

  • Create a Turtle file

    The final .csv table contained more then 22K entries, which were then linked as a RDF file using the Turtle syntax. The final Turtle file contained more than 200K relations, linking records with their descriptor and the related artists with (some of) their Wikidata properties.

  • Graph Embedding

    Once the linked dataset was built, it became possible to do some interesting operations on it. Pykeen was used as a library to create a graph embedding of the dataset. In order to exploit the Pykeen functionalities, the graph was converted into a tabular file, each row representing a triple. This allowed to perform many significant operations on the data, such as link predicition and sentimental analysis.
